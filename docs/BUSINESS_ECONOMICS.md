# Бизнес-логика финансовых показателей

Интерфейс менеджера показывает рабочий список бронирований, отсортированный по ожидаемой денежной потере. Экономическая оценка действий менеджера вынесена в отдельный раскрывающийся блок и не используется как главный KPI продукта.

## Основные показатели

**Сумма всех броней** — сумма потенциальной выручки по всем загруженным бронированиям.

```text
Сумма брони = цена за ночь × количество ночей
```

**Ожидаемые потери** — сколько денег отель в среднем может потерять из-за отмен, если ничего не делать.

```text
Ожидаемые потери = калиброванная вероятность отмены × сумма брони × тарифный коэффициент потери
```

**К проверке** — бронирования с высоким/критическим риском и существенной ожидаемой денежной потерей. Это рабочая очередь менеджера.

**Потери в списке** — ожидаемые потери только по броням, которые попали в рабочий список.

**Доля риска в списке** — какая часть всех ожидаемых потерь сосредоточена в рабочем списке. Этот показатель показывает, насколько хорошо список концентрирует денежный риск.

## Оценка действий менеджера

В раскрывающемся блоке можно задать два параметра:

```text
Процент потерь, который можно предотвратить
Стоимость проверки одной брони
```

После этого сервис показывает:

```text
Можно сохранить = потери в списке × процент предотвращенных потерь
Оценка действий = можно сохранить − стоимость проверки
```

Это сценарная оценка. Она не является метрикой качества модели и не должна заменять рабочий список к проверке.

## Почему `Non Refund` не формирует потери

Модель оценивает вероятность отмены для всех тарифов. Денежная очередь менеджера строится не по одной вероятности отмены, а по ожидаемой потере. Для `Non Refund` потеря выручки равна нулю, потому что оплата не возвращается. Для `Refundable` и `No Deposit` ручного понижения нет: модель сама оценивает вероятность отмены.

## Актуальная формула денежного риска

Денежный показатель считается не через сырой `risk_score`, а через калиброванную вероятность отмены:

```text
expected_loss = cancellation_probability × booking_value × deposit_loss_factor
```

Где `cancellation_probability` — вероятность после Platt-калибровки на validation split. `risk_score` остается техническим score модели и не используется напрямую для денежной оценки.

Коэффициент `deposit_loss_factor`:

```text
No Deposit  = 1.00
Refundable  = 1.00
Non Refund  = 0.00
```

`Refundable` не дисконтируется вручную. `Non Refund` не формирует ожидаемую потерю, потому что деньги невозвратные.
